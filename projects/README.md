# Projects

This lane is for end-to-end ownership.

Each serious project should earn its own folder with:

- `README.md` - problem, approach, setup, results, and next steps
- `src/` or app code
- `data/` or a clear data-access note
- `runs/` or experiment summaries when useful
- `assets/` for local screenshots, demo clips, and diagrams

## Project Bar

A project is portfolio-worthy when it has a working demo, a clear technical story, evidence of iteration, and a polished explanation that a technical reviewer can scan in under five minutes.
